# 模块规格说明：Quant Intelligence

**状态**：主真相层草案  
**对应实现**：`src/openclaw_trader/modules/quant_intelligence/`  
**来源承接**：`001`、`004`

## 1. 背景与目标

`quant_intelligence` 负责把交易所标准化事实转成结构化市场判断，但它不再承担建议器职责。它输出的是可被策略、风控和 Agent 共同消费的市场事实，而不是动作建议。

## 2. 职责

- 计算 `1h`、`4h`、`12h` 多时域判断
- 输出 regime、方向、置信度和诊断信息
- 维持量化产物可复现、可比较、可校准

## 3. 拥有资产

- `CoinForecast`
- `HorizonSignal`
- 多时域诊断、regime 和质量指标

## 4. 输入

- `Trade Gateway.market_data` 输出的结构化市场事实
- 模型文件、校准参数和运行期特征配置

## 5. 输出

- 面向策略和风控的结构化市场事实
- 多时域诊断与辅助解释

## 6. 直接协作边界

- 向 `policy_risk`、PM、RT 和 `agent_gateway` 运行时输入层提供结构化市场事实
- 向 `memory_assets` 提交可追溯的量化产物引用

## 7. 不负责什么

- 不负责硬风控规则和事件记忆
- 不负责直接给出建议、批准或拒绝动作
- 不负责交易所接入细节和执行交付

## 8. 当前已定

- `1h / 4h / 12h` 都保留为结构化市场事实
- `PM` 与 `Risk Trader` 当前都可以把 `1h / 4h / 12h` 作为信息参考源
- `policy_risk` 仍主用 `4h / 12h`，不把 `1h` 纳入主判断链
- 输出必须保持“事实性”，不得混入隐式建议
