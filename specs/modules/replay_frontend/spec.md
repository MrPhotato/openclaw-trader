# 模块规格说明：Replay Frontend

**状态**：主真相层草案  
**对应实现**：`src/openclaw_trader/modules/replay_frontend/`  
**来源承接**：`001`、`007`

## 1. 背景与目标

`replay_frontend` 负责把统一读模型和事件流变成可观测界面。它不直接读取散乱日志文件作为主接口，而应建立在 `memory_assets` 和结构化事件协议之上。

## 2. 职责

- 提供回放查询入口和前端订阅面
- 呈现模块协作时间线、状态和结果
- 服务运维、审计和复盘

## 3. 拥有资产

- 回放查询模型
- 前端订阅与渲染提示

## 4. 输入

- `memory_assets` 读模型
- 结构化事件流

## 5. 输出

- 时间窗口回放
- 模块状态与策略版本可视化
- 前端订阅/查询接口

## 6. 直接协作边界

- 依赖 `memory_assets` 提供统一读面
- 消费统一事件协议而非私有日志

## 7. 不负责什么

- 不定义业务真相
- 不直接读取零散 runtime 日志文件作为主来源

## 8. 当前已定

- 前端只消费统一读模型和事件流
- 回放、审计和可视化基于结构化协议

## 9. 待后续讨论

- 实时事件流与静态查询的组合方式
- 页面、组件和交互层级
