# 任务分解：系统模块化蓝图与现状全拆解

**功能分支**：`codex/001-system-blueprint`  
**规格文档**：`specs/001-system-blueprint/spec.md`

## 第一波：现状拆解与事实归档

- [ ] T001 固化当前系统基线文档，包含主运行路径、真相源顺序、现有模块能力、入口点与主要痛点。
- [ ] T002 建立“当前代码文件 -> 未来 10 模块”映射清单，并标记哪些文件跨边界、哪些未来需要拆分。
- [ ] T003 盘点当前 CLI、FastAPI、OpenClaw、通知与运行态文件接口，形成统一接口现状表。
- [ ] T004 形成当前 live 运行态复现手册，覆盖本地配置、运行态目录、状态库、报表、部署进程与 OpenClaw workspace。

## 第二波：目标架构与模块契约

- [ ] T005 定义 10 个顶层模块的职责、端口、数据所有权、非职责与事件输出要求。
- [ ] T006 定义 3 条跨域平面的协议：结构化事件与日志、RabbitMQ、参数治理。
- [ ] T007 定义 4 个 Agent 的职责边界、自主权限、升级与回执规则。
- [ ] T008 定义统一状态机、统一外部主动触发入口、回放链路与前端观察视图。
- [ ] T009 形成 OpenClaw 独立适配契约、事件 envelope 契约、RabbitMQ topic 契约、控制接口契约。
- [ ] T010 补齐量化小模型训练 / 推理 / artifact / 阈值与使用方式的复现手册。

## 第三波：迁移路径与质量门禁

- [ ] T011 制定分波次迁移方案：骨架波次、收口波次、智能体波次、可视化波次。
- [ ] T012 为每一波次定义可验证的完成条件、回滚策略与运行态兼容要求。
- [ ] T013 建立蓝图质量检查清单，确保模块边界、真相源、状态机、消息协议和前端观测链路一致。
- [ ] T014 将后续具体实现拆成可执行 issue / task 队列，避免继续以补丁形式推进。
