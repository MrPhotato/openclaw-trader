# 迁移波次

## 波次 1：骨架波次

目标：先建立以后所有模块都要依赖的公共骨架。

交付：

- 统一事件 envelope
- RabbitMQ 基础拓扑
- 参数治理模型
- 显式状态机骨架
- 外部统一主动触发入口

## 波次 2：收口波次

目标：把现有代码按边界收口，但尽量不动核心交易语义。

交付：

- 把 `dispatch/__init__.py` 拆成状态机与编排器 + 多智能体协作网关 + 通知服务适配
- 把 `strategy/__init__.py` 拆成策略与组合意图 + 上下文构建 + 状态与记忆管理
- 把 `perps/runtime/__init__.py` 拆成量化结果消费 + 风控守卫 + 执行桥接

## 波次 3：智能体波次

目标：从单一 `crypto-chief` 过渡到 4 Agent 协作。

交付：

- PM
- Risk Trader
- Macro & Event Analyst
- Crypto Chief 统筹层
- 升级、回执、Learning 流程

## 波次 4：交付波次

目标：让系统变成可实时观察、可回放、可调参的平台。

交付：

- 回放与前端模块
- 参数调节面板
- 通知服务统一入口
- 历史轨迹 / replay 视图

## 每一波次都必须满足的条件

- 不破坏当前 live 路径可理解性
- 保留回滚点
- 所有新增边界都要伴随结构化事件
- 文档先于实现更新
