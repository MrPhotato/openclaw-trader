# 功能规格说明：市场智能与风险守卫

> **迁移说明（2026-03-15）**：`Trade Gateway`、`quant_intelligence` 与 `policy_risk` 的主规格现位于 `specs/modules/`。本文件继续保留为横切的市场事实与风险 feature 文档。

**功能分支**：`codex/004-market-intelligence-guards`  
**创建日期**：2026-03-11  
**状态**：草案  
**输入描述**：定义交易网关 `market_data` 子域、新闻事件、多时域量化判断和风控守卫的输入输出契约，服务新的永续交易主路径。

## 1. 背景与目标

`001` 已明确当前 live 主路径依赖市场快照、新闻事件和 `1h/4h/12h` 量化判断。`004` 的目标是把这条事实链路单独写成可实现 feature，使后续策略、Agent 和执行只消费结构化结果，而不是再直接读原始外部源。

本 feature 只围绕新的永续主路径，不兼容旧 spot 路径。

## 2. 当前系统基线

- 当前市场输入分散在交易所接入、量化特征和新闻目录中。
- 当前量化框架已经稳定为 `1h/4h/12h` 多时域预测。
- 当前第一批收敛目标是：
  - `1h/4h/12h` 都保留为结构化市场事实
  - `1h` 当前恢复为 PM 与 RT 的参考信息源，但不进入 `policy_risk` 主判断链
- 当前风控守卫第一批目标已经从复杂软建议体系收敛为硬风控边界。
- 本 feature 依赖 `002` 的事件协议和 `003` 的控制平面，但不进入策略意图和执行模块。

## 2.2 2026-03-12 边界调整

原先独立的“数据接入与标准化模块”不再作为顶层模块存在。

本 feature 后续负责的是：

- `Trade Gateway` 内部 `market_data` 子域输出的标准化市场与账户事实
- 新闻事件输出
- 量化判断输出
- 风控与执行守卫输出

也就是说，`004` 关注的是“事实读取与判断链”，但交易所接入边界在顶层上已统一归入 `Trade Gateway`。

## 2.1 2026-03-11 收敛决议

经过对当前实现与 `codex/dev` 的对比，决定收敛 `policy_risk` 的职责，避免系统把模型输出误用为二级策略器。

本轮已经确认的方向：

- `quant_intelligence` 只提供结构化市场事实，不直接给出软性或硬性的交易建议。
- `1h/4h/12h` 都作为量化市场事实保留；其中 `1h` 当前仅恢复给 PM 和 RT 做参考，不进入 `policy_risk` 主判断链。
- `policy_risk` 只保留硬风控，不再保留复杂的软引导和多层降级逻辑。
- 事件、组合、模型不确定性只保留为诊断信息，不再直接生成复杂的建议型输出。

当前 `policy_risk` 的目标输出只保留四类信息：

1. 交易可用性边界：当前是否允许交易。
2. 硬暴露边界：最大杠杆、最大总敞口、最大单币仓位、最大单笔下单。
3. 持仓风险边界：`observe / reduce / exit / cooldown / breaker`。
4. 诊断信息：最近事件标题、`4h/12h` 摘要、暴露快照；不包含建议型字段。

### 2.1.1 2026-03-13 已确认的最小硬风控参数

本轮已经确认第一版硬风控只保留以下参数和状态，且以 `UTC 00:00` 作为每日分隔点：

- `risk_limits`
  - `max_leverage = 5.0`
  - `max_total_exposure_pct = 100.0`
  - `max_symbol_position_pct = 66.0`
  - `max_order_pct = 33.0`
- `position_risk_state`
  - `normal`: `drawdown < 4.0%`
  - `observe`: `4.0% <= drawdown < 7.0%`
  - `reduce`: `7.0% <= drawdown < 10.0%`
  - `exit`: `drawdown >= 10.0%`
- `cooldown`
  - 仅在 `position_risk_state = exit` 后触发
  - 单币持续 `30` 分钟
- `panic_exit`
  - 定义为“当日账户权益相对当日 `UTC 00:00` 基准减少 `15.0%`”
- `breaker`
  - 触发条件一：当日发生 `1` 次 `panic_exit`
  - 触发条件二：同一 `UTC` 日内发生 `2` 次 `position_risk_state = exit`
  - 生效后持续到次日 `UTC 00:00`
  - 允许人工提前解除，或人工无限延长，直到下次指令

### 2.1.2 2026-03-13 持仓回撤计算约束

`position_risk_state` 的回撤计算必须基于“当前持仓的单位金额表现”或等价的单位持仓价格路径，不能直接使用持仓总名义金额回撤。

目的：

- 避免部分减仓后因当前名义金额下降，回撤百分比被错误放大
- 保证 `observe / reduce / exit` 反映的是该持仓本身的价格不利变化，而不是仓位规模变化

实现要求：

- 持仓缩量不得单独导致回撤百分比跃升到更高阶段
- 只有价格路径、方向变化、重新建仓或持仓身份重置，才能重置或改变该回撤基准

## 3. 用户场景与验收

### 场景 1：策略和执行层需要稳定的市场结构化输入

后续模块需要固定输入，而不是自己再去决定行情字段、新闻摘要和 horizon 结果如何组织。

**验收标准**

1. 必须定义标准化市场快照和新闻事件实体。
2. 必须定义多时域预测和风险守卫输出实体。

### 场景 2：实现者需要保持当前永续主路径语义

实现者在重构时，不能随意改动当前量化职责分层和硬风控边界语义。

**验收标准**

1. 文档必须明确 `12h/4h/1h` 的职责分层与第一批采纳边界。
2. 文档必须明确 `policy_risk` 只输出硬风控对象，诊断信息不得回流为建议。

## 4. 功能需求

- **FR-001**：系统必须定义 `MarketSnapshotNormalized`，覆盖价格、funding、premium、open interest、volume 等标准化市场事实，并明确其来源于 `Trade Gateway.market_data`。
- **FR-002**：系统必须定义 `NewsEventMaterialized` 和 `NewsDigestReady`，将原始新闻源转为结构化事件和摘要。
- **FR-003**：系统必须定义 `MultiHorizonPredictionReady`，至少包含 `1h`、`4h`、`12h` 的方向、概率、regime 和诊断。
- **FR-004**：系统必须定义 `RiskGuardDecisionReady`，覆盖 `trade_availability`、`risk_limits`、`position_risk_state` 和诊断信息。
- **FR-005**：文档必须明确当前有效职责：`1h/4h/12h` 都作为市场事实输入；其中 `1h` 当前只面向 PM 与 RT 参考，不进入 `policy_risk` 主判断链。
- **FR-006**：本 feature 不得定义策略意图、执行判断、执行计划或通知语义。
- **FR-007**：`Trade Gateway.market_data` 第一批必须补齐多尺度价格序列、价格形态、关键价位、突破/回踩、波动状态、最近订单/成交历史等采集和标准化入口，但不规定这些字段的消费方式和存储方式。
- **FR-008**：`risk_limits` 第一版必须使用已确认值：`5.0 / 100.0 / 66.0 / 33.0`。
- **FR-009**：`position_risk_state` 的阶段阈值第一版必须使用 `4.0 / 7.0 / 10.0`。
- **FR-010**：`cooldown` 仅由 `position_risk_state = exit` 触发，持续 `30` 分钟。
- **FR-011**：`panic_exit` 必须定义为当日账户权益相对当日 `UTC 00:00` 基准下降 `15.0%`。
- **FR-012**：`breaker` 必须在“当日 `1` 次 `panic_exit`”或“同一 `UTC` 日内 `2` 次 `position_risk_state = exit`”后激活，并默认持续到次日 `UTC 00:00`，同时支持人工解除和人工延长。
- **FR-013**：`position_risk_state` 的回撤计算不得直接基于持仓总名义金额变化，必须避免减仓导致的虚假回撤放大。

## 5. 非功能要求

- **NFR-001**：所有输出实体都必须可被策略、Agent 和前端共同消费，不依赖内部代码对象。
- **NFR-002**：市场、新闻、量化和风险守卫之间的接口必须稳定，避免后续 feature 重复解释字段含义。
- **NFR-003**：文档必须围绕新的永续主路径，不再为旧现货兼容路径保留实现责任。

## 6. 关键实体

- **MarketSnapshotNormalized**：标准化市场事实。
- **MarketContextNormalized**：价格形态、关键价位、波动状态、执行历史等派生市场上下文。
- **NewsEventMaterialized**：结构化新闻/事件对象。
- **MultiHorizonPredictionReady**：多时域预测结果。
- **RiskGuardDecisionReady**：硬风控边界对象，不包含建议型动作输出。

## 7. 假设与约束

- 外部市场源可在实现阶段替换，但标准化实体不变。
- 本 feature 只定义新系统需要的输入输出，不复刻旧 spot 采样与旧接口细节。
- 当前量化模型框架沿用 `001` 和 `12-quant-model-reproduction.md` 的职责分层，不在本 feature 中重新拍板。
- 第一批允许 `1h` 进入 PM/RT 的上下文参考层，但它不得进入 `policy_risk` 主判断链，也不得直接重写执行/风控硬规则。

## 8. 成功标准

- **SC-001**：`005` 可以只消费 `004` 定义的结构化市场与风险输出，而不再读原始外部源。
- **SC-002**：`006` 可以只消费 `004` 定义的信息源视图，而不再自己解释原始新闻和量化输出。
- **SC-003**：当前永续主路径的量化职责和硬风控边界在文档中被稳定表达，没有悬空解释。
